# Flutter command catalog

Quick reference for common Flutter CLI commands in this repo. Run all commands from the repository root (`retentio-frontend/`, where `pubspec.yaml` lives).

🌐 English | [中文](flutter_commands_zh.md)

## Devices

List connected emulators, simulators, and physical devices:

```bash
flutter devices
```

| Command | Purpose |
| ------- | ------- |
| `flutter doctor` | Check toolchain and platform setup |
| `flutter emulators` | List available emulators |
| `flutter emulators --launch <emulator_id>` | Start an emulator |

If no device appears, start an iOS Simulator or Android emulator, or connect and authorize a physical device.

## Run

Default debug run (picks a device if only one is available):

```bash
flutter pub get
flutter run
```

Run on a specific device (`-d` accepts device ID or name from `flutter devices`):

```bash
flutter run -d <device_id>
flutter run -d "iPhone 16 Pro"
flutter run -d emulator-5554
```

| Flag | Meaning |
| ---- | ------- |
| `--release` | Flutter release build mode (optimized, no debug tooling) |
| `--dart-define=API_ENV=release` | Production API host (see below) |
| `--dart-define=API_HOST=<url>` | Override API base URL (takes precedence over `API_ENV`) |

### Common combinations

```bash
# Dev API (default for debug run) on a chosen device
flutter run -d <device_id>

# Production API on a chosen device (still debug build)
flutter run -d <device_id> --dart-define=API_ENV=release

# Release build + production API
flutter run -d <device_id> --release --dart-define=API_ENV=release

# Local backend on Android emulator
flutter run -d emulator-5554 --dart-define=API_ENV=debug

# Custom host (e.g. physical device on LAN)
flutter run -d <device_id> --dart-define=API_HOST=http://192.168.1.10:8080
```

## Build

```bash
# Android
flutter build apk --release --dart-define=API_ENV=release
flutter build appbundle --release --dart-define=API_ENV=release

# iOS
flutter build ipa --release --dart-define=API_ENV=release
```

Point a release/IPA build at a non-production API when needed:

```bash
flutter build ipa --dart-define=API_ENV=dev
```

## Backend environment (`API_ENV`)

Compile-time API host selection via `--dart-define`. Implementation: `lib/services/env.dart`.

| `API_ENV` | Host |
| --------- | ---- |
| `debug` | `http://localhost:8080` |
| `dev` | `https://10.0.0.145:8443` |
| `release` | `https://api.retentio.app:8443` |

When `API_ENV` is omitted:

- **`flutter run`** and other non-release builds default to **`dev`**
- **`--release`** / product builds (`flutter build ipa`, Xcode Archive) default to **`release`**

`API_HOST` overrides the mapping when set to a non-empty URL.

## iOS notes

- `ios/Flutter/Generated.xcconfig` is generated by Flutter. If Xcode reports it missing, run `flutter pub get` before opening `ios/Runner.xcworkspace`.
- If plugins fail with `Flutter/Flutter.h` file not found, run `flutter pub get` then `cd ios && pod install` on the build Mac.
